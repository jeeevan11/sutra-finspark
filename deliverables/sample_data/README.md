# SUTRA sample data (seed 42)

Synthetic 24h of a mid-size bank's telemetry, generated by
`backend/sutra/generator/`. No real PII — all IDs are obviously synthetic.

- `security_telemetry.jsonl` — 44,902 events: auth_login, edr_alert, tls_session,
  payee_added, password_reset (one JSON object per line).
- `transactions.jsonl` — 56,233 core-banking transactions.
- `labels.json` — generator ground truth: which events belong to attack scenarios
  A (account takeover + structuring), B (compromised terminal), C (HNDL exfil) vs
  benign noise vs borderline single-signal lookalikes.

This is the sample data referenced in the demo video. Regenerate with
`make export-sample-data`.
